# 📂 [07] 사용자 피드백 루프 (Human-Feedback Loop)

Archie-G는 사용자의 수정을 최종 권위(Source of Truth)로 인정하고 이를 통해 지역 지식을 고도화합니다.

---

### 1. 피드백 파일 관리
*   **분석 초안:** 모든 AI 생성 분석 초안은 `analysis_drafts/` 폴더 내에 별도로 생성하여 보관한다.
*   **파일 명명:** `[파일명]_v[버전].draft.md` 형식을 권장한다.

### 2. 수정 사항 반영 (Feedback-In)
*   사용자가 초안을 직접 수정한 경우, 해당 파일은 `[FEEDBACK]` 태그와 함께 에이전트에게 다시 입력되어야 한다.
*   에이전트는 사용자의 수정된 텍스트와 다이어그램을 분석하여 자신의 이전 오류나 누락 사항을 파악한다.

### 3. 지식 업데이트 및 자산화
*   **예외 규칙 기록:** 사용자가 특정한 로직 흐름이나 아키텍처 해석을 수정한 경우, 에이전트는 이를 '커스텀 예외 규칙'으로 로컬 벡터 DB(ChromaDB 등)에 기록한다.
*   **우선순위:** 이후 동일하거나 유사한 모듈을 분석할 때, 에이전트는 원본 코드의 추론 결과보다 **사용자의 과거 수정 이력(Feedback History)을 최우선 참조**하여 분석을 수행한다.
*   **지속적 학습:** 피드백이 누적될수록 에이전트는 해당 프로젝트에 특화된 아키텍처 스타일과 비즈니스 로직을 학습하게 된다.
